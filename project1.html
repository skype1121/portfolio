진행 기록

STEP 01
하드웨어 초기 검증 (Prototype)
[ PROBLEM ] 다관절 기구적 설계 결함 감지 및 치수 확인 불가
[ FAILURE ] 시각적 CAD만 의존 시 모터 간 간섭 및 충돌 등 하드웨어 사고 우려
[ FIX ] 저비용 3D 프린팅으로 1차 프로토타입 출력 및 실물 가조립 평가
[ RECORD_LOG ]
초기 설계 단계에서 부품 간섭과 모터 체결 부위를 파악하기 위해, 프로토타입 다리 관절을 3D 프린터로 대충 출력하여 물리적 치수와 역학적 구동 범위를 1차 검증했습니다.
3D Prototype Dimension Check
STEP 02
SW 인프라 & 데이터 파이프라인 (ROS2)
[ PROBLEM ] 장비 간 의존성 문제 및 비표준 통신 규격
[ FAILURE ] 상태 값 모니터링 누락 시 즉각적인 디버깅 불가
[ FIX ] ROS2 Topic을 통한 노드 간 상태/제어 데이터 통신 루프 최적화
[ RECORD_LOG ]
Docker 컨테이너 내에서 ROS2 Humble 워크스페이스를 패키징하고, 실제로 각 모터 노드와 센서 노드를 묶어 실시간으로 전송되는 로스(ROS) 토픽을 디버깅/구독(Subscribe)하는 파이프라인이 완성되었습니다.
ROS2 Humble Topic Subscribe Docker
STEP 03
시뮬레이션 (Isaac Sim & MuJoCo)
[ PROBLEM ] 하드웨어 파손 위험 및 학습 데이터 절대적 부족
[ FAILURE ] 실물 구동 직진입 시 반복적인 모터 소모 및 기구부 파손 발생
[ FIX ] 가상 가상환경 모델링(MJCF) 후 로봇 다리(Leg) 동작 무한 검증
[ RECORD_LOG ]
SolidWorks 모델을 MJCF로 정밀하게 변환하여 가상 컴퓨팅 환경 속에 불러온 뒤, 실제 로봇 다리가 움직여야 할 Stand-up 및 보행 파라미터 연산의 물리적인 거동 가능성을 안전하게 테스트하고 있습니다.
Isaac Sim MuJoCo Virtual Validation
STEP 04
실시간성 최적화 (Real-time Tuning)
[ PROBLEM ] OS 소프트웨어 기반 무작위 지연(Latency)
[ FAILURE ] 10ms 통신 지연 누적 시 로봇 고속 제어 발산 발생
[ FIX ] CPU isolation & RT-Scheduler 적용하여 14µs Latency 보장
[ RECORD_LOG ]
운영체제 단에서 커널 레이턴시를 제어하기 위한 실시간성(RT) 튜닝을 거쳐 안정적인 1kHz 제어 루프를 방어했습니다.
CORE INTEGRATION
STEP 05
시스템 통합 (Hardware Integration)
[ PROBLEM ] 시뮬레이션과 실제 물리 구동 사이의 극심한 제어 기술 괴리
[ FAILURE ] 물리 모터 ROS2 연동 부재로 인한 명령 하달 실패
[ FIX ] AK70 CAN 드라이버 네이티브 통합 및 실물 구동 검증 완료
[ RECORD_LOG ]
AK70 모터를 제어하기 위한 SocketCAN 기반 C++ 드라이버 로직을 래핑하고, ROS2 노드 환경에서 직접 명령을 보내 물리 하드웨어가 의도치 않은 떨림 없이 동작하는 성과를 이끌었습니다.
AK70 CAN Driver Hardware Test Success
STEP 06
시각 인지 및 센서 융합 (D455 Vision)
[ PROBLEM ] 운동학 파라미터만으로는 동적 지형 장애물 파악의 근본적 불가
[ FAILURE ] 전면부 공간 인지 누락으로 인한 로봇 주행성 100% 붕괴 위험
[ FIX ] Intel RealSense D455 심도(Depth) 카메라 연계 및 입체 공간 인지
[ RECORD_LOG ]
비전 모델과 물리 제어의 결합을 준비하며 Intel RealSense D455 뎁스 카메라로 깊이 정보를 추출하고 공간을 인지하는 과정을 직접 확인하고 도입했습니다.
RealSense D455 Depth Vision Embodiement
